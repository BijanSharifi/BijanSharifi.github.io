NFA to DFA and Regular Expression Generator (OCaml) (School) : An NFA has the following fields to it: sigma (alphabet), state list, starting state, final state(s), and delta (transitions). This program performs the algorithm to turn any NFA into a DFA. Function is written by using a set of helper functions that help break this problem down into smaller parts. Some of the helper functions include move, eclosure, and finding new states and transitions as the subset construction steps. A regular expression can also be defined in 5 "fields" as well. A regular expression contains some combination of the following: empty string, chars, unions, concatination, and Kleene star. This function converts a regular expression to an NFA, because any regex can be represented by an NFA.
Lexer, Parser, and Interpreter (OCaml) (School) : A program that creates a top-level interpreter for the OCaml Language. Starts off by creating a tokem list of all the inputed words to make sure they are valid, then uses the parser to make sure it gramatically makes sense, then finally gets evaluated using the interpreter to make sure the meaning of the line makes sense.
